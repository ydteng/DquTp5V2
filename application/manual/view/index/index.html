前言说明：

        1.所有订单超过24小时都将失效，不能进行接单，确认订单，取消订单操作。

       （前端需要对状态码做判断，进行限制性操作，状态码为“1000”“5000”“5001”“6000”的订单不能进行删除订单以外的其他操作。此情况一般不会发生，但为了防止他人恶意发送数据而在服务器做的防护）

        2.不能把状态码为“1000”“5000”“5001”“6000”的订单的发单人信息显示给接单人。

        3.只有状态码为“3000”“4000”“4001”的订单才能有确认订单操作

状态码 status 释义：

1000.失败（订单人接取）
1001.订单取消

2000.等待接单

3000.已接单,派送中

4000.接单方已确认订单
4001.发单方已确认订单

5000.接单方24小时后未确认订单，订单自动完成
5001.发单方24小时后未确认订单，订单自动失效

6000.最终状态不做判断，双方都确认订单

3.用户信息类

  • 添加个人信息及收货地址（方法：post）

https://camserv.cn/api/v1/address

说明：token在header中传入，其他在body中传入，json格式。

参数值
字段 释义
token 令牌（在header中传入）
nickname 昵称（区别于微信名）
real_name 真实姓名（保留，默认自定义）
mobile  手机号码
province_id 省份ID
school_id 学校ID
 detail 详细地址
返回值
字段 释义
code http状态码
msg 成功返回“ok”（主要）
errorCode 错误码

获取用户地址信息（方法：get）

https://camserv.cn/api/v1/address

说明：区别于上面的添加地址，接口地址相同，但是方法不同。token还是在header中。

参数值
字段 释义
token 令牌（在header中传入）
返回值
字段 释义
id 标记性id，可不使用
user_id 标记性id，可不使用
nickname 昵称
real_name 真实姓名
mobile 电话号码
detail 详细地址
province name 省份名称
school name 学校名称

  • 获取个人订单信息（方法：get）

https://camserv.cn/api/v1/order/:page

说明：获取个人订单列表，返回的事按时间顺序从降序排序（即从晚到早）。token在header中传入

参数值
字段 释义
page 是用来分页用的id，防止一次性加载数据太多，每页返回10个数据
返回值
字段 释义
id 订单id，可用于查看订单详情
status （重要）订单状态，代码表示：见手册开头
cost 订单费用
start_point 订单起点
end_point 订单终点信息
item_type 订单物品类型（如快递，餐饮等）
order_num 订单号

  • 接取订单（方法：post）

https://camserv.cn/api/v1/order/pack

说明：需要在header中传入token，订单id需要在body中传入。前端需要判断接取的是否是自己订单

限制：1.接取订单后才能看发单人的地址信息

           2.不能接取自己发的订单

参数值
字段 释义
id 订单id
返回值
字段 释义
id 订单id，可用于查看订单详情
status （重要）订单状态，代码表示：见手册开头
cost 订单费用
start_point 订单起点
end_point 订单终点信息，包括了个人信息，需要显示给接单人
item_type 订单物品类型（如快递，餐饮等）
order_num

订单号
detail 订单详情

  • 取消订单（方法：post）

https://camserv.cn/api/v1/order/cancel

说明：需要在header中传入token，订单id需要在body中传入

限制：1.已被接取的订单，发单方无法取

参数值
字段 释义
id 订单id
返回值
字段 释义
msg 成功：   取消成功   ；   失败：取消失败或其他异常
